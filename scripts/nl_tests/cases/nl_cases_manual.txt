# Canonical curated set for natural-language manual cases.
#
# Format: suite|name|tags|prompt|expect=<substring>
#   - 5th field is OPTIONAL. Whenever present, the runner asserts that the
#     final response text contains the literal substring AND status=succeeded.
#     Missing or failed → assertion=fail in the summary.
#   - tags is comma-separated. Special tags:
#       chat_force   — submit as kind=run_skill skill_name=chat (bypasses
#                      intent_router; the only way to actually exercise the
#                      builtin chat skill, since intent_router will short-circuit
#                      most chat prompts via direct_reply_candidate).
#       natural      — phrased loosely, tests model robustness to fuzzy intent.
#       cn           — Simplified Chinese prompt.
#       missing_file_graceful / write_and_deliver — legacy assertion hints used
#                      by upstream regression tooling; keep as-is.
#
# Design notes (rev 2026-04-17):
#   - Removed 15+ near-duplicate `act_xxx` vs `natural_xxx_cn` pairs (legacy
#     file kept at nl_cases_manual.legacy.txt for reference).
#   - Promoted 12 cases to hard assertions via `expect=...`.
#   - Added explicit chat_force cases (B5) — the legacy file had ZERO cases
#     that actually entered the chat skill code path.
#   - Added more failure cases (B4): missing_dir, outside_workspace.
