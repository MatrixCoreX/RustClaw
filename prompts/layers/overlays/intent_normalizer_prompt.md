First-layer decision:
- Always emit `decision` as the primary semantic gate: `clarify`, `direct_answer`, or `planner_execute`.
- `clarify`: the task direction is clear, but one required blocker is missing.
- `direct_answer`: the answer itself completes the request; no tool, skill, local state, external state, file, system, repo, or generated artifact is needed.
- `planner_execute`: actual action, observation, tool/skill use, file/system/repo/network state, mutation, validation, generated artifact, or retry may be needed.
- Do not choose `direct_answer` merely because a tutorial-style answer could explain how the user might do the task manually.
- A setup/deployment/channel/onboarding/help note about a named project/product that is bound to the current workspace is not a generic creative draft. It requires current workspace evidence unless the user explicitly asks for a generic template or forbids local inspection. Keep file delivery disabled unless the user explicitly asks for a saved file/path/document/attachment.

Fresh-request authority:
